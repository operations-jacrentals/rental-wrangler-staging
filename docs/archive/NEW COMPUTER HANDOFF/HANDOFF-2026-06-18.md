### 1 · Blued Steel theme suite (PRs #60/#61/#69/#72/#74/#77)

The full card surface is now a cohesive blued-steel data-plate across all three columns.

- **Search bar / `.searchwrap`** — metal plate treatment matching the cards: gradient overlay +
  `tex-metal-blued.jpg` tiled at 300px, blued border + inset glow. Previously the search bar looked
  like an orphaned flat input.
- **List cards** (`[data-theme="bluedsteel"] .col > .card`) — `tex-metal-blued.jpg` at 340px tile,
  blued-teal gradient overlay. Card cap hazard stripe is now yellow (`--stripe: var(--yellow)`).
- **Standard-view sections** (`.section`) — milled-in recess panel: `rgba(11,15,24,.36)` + inset top
  glow + drop shadow. Sections read as wells machined into the plate surface.
- **Rows** (`.row`) — same milled-panel recess formula. Row borders use `rgba(150,178,222,.16)`.
- **Anchored cards** — same plate formula, consistent with non-anchored state.

**The milled-panel recess formula** (for any dark panel surface in bluedsteel):
```css
background: rgba(11,15,24,.36);
box-shadow: inset 0 1px 0 rgba(150,178,222,.12), 0 2px 8px -3px rgba(0,0,0,.55);
```

Full card rule in `style.css` (`[data-theme="bluedsteel"] .col > .card`):
```css
[data-theme="bluedsteel"] .col > .card {
  position: relative;
  background:
    linear-gradient(180deg, rgba(58,80,118,.34), rgba(8,11,18,.68)),
    url('assets/tex-metal-blued.jpg');
  background-size: cover, 340px;
  background-repeat: no-repeat, repeat;
  --stripe: var(--yellow, #f5c542);
}
```

### 2 · Back button → list view fallback (PR #82)

`cardBack()` previously did nothing when `backStack` was empty. Now:
- Empty stack + card in `standard` mode + record open → navigates to list view (same as clicking
  the list tab). The lost state is pushed onto `fwdStack` so Forward still works.
- `cardJog()` shows the Back `←` button even with an empty stack when in a record — users always
  see a visible "go back" affordance.

This closes the dead-end: "I opened a record from search, now I'm stuck with no Back button."

Code: `cardBack(card)` and `cardJog(card, cs)` in `app.js`.

### 4 · Column polish — toggle gap + center column texture (PR #99)

Two visual fixes shipped together:

**Toggle gap equalized.** The gap above the card tab bar was visually smaller than below. Root cause:
the 6px hazard stripe is `position: absolute` at the card top, eating into top margin. Fixed by
`margin-top: 13px` on `.card > .tabrow` (formula: `desired_gap + stripe_height - border = 8 + 6 - 1`).
Now exactly **8px above, 8px below** on all three columns.

```css
.card > .tabrow {
  margin: 13px 8px 8px;   /* was 9px 8px 5px — formula: 8+6-1=13 for top, equal visual gap */
}
```

**Center column texture flip.** The left and right columns both tiled `tex-metal-blued.jpg`
identically. A new asset `tex-metal-blued-flip.jpg` (horizontal mirror via PowerShell
`RotateNoneFlipX`) is applied to the middle column only. The grain direction differs enough that
frequent users don't register the same pattern repeating. The flip file lives at
`assets/tex-metal-blued-flip.jpg` (~168KB).

```css
[data-theme="bluedsteel"] .col[data-col="middle"] > .card,
[data-theme="bluedsteel"] .col[data-col="middle"] > .card.anchored {
  background:
    linear-gradient(180deg, rgba(58,80,118,.34), rgba(8,11,18,.68)),
    url('assets/tex-metal-blued-flip.jpg');
  background-size: cover, 340px;
  background-repeat: no-repeat, repeat;
}
```

## How to work safely

- **Branch → PR → squash-merge** always. `main` is branch-protected (required `smoke` CI check).
- **Gates:** `node ci/smoke.mjs` · `node ci/logic-test.mjs` · `node ci/gen-rule-usage.mjs --check`
- **Port 8000 is reserved on this machine.** CI hardcodes 8000. Swap ports before running:
  `sed -i 's/8000/9147/g' ci/smoke.mjs ci/logic-test.mjs`, run gates, then `git checkout -- ci/`
- **`wait-merge` syntax:** `node rw-automation/gh-api.mjs wait-merge <branch-name> <pr-number>`
  — both args are required. Branch name is the git ref for check-run lookup; PR number is for merge.
- **KPI mask:** leave the blur block alone until Jac explicitly says to remove it.
- **Backend `Code.gs`** — never committed. `backend/` is gitignored. Deployed via clasp at
  version @17 (as of this session). Re-auth each session: `clasp login --no-localhost`.

- Never push directly to `main` — always branch → PR.
- Do **NOT** remove the KPI blur mask until Jac explicitly instructs it.
