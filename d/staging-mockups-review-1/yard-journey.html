Yard Journey is now its own Units section, not a header-strip rail — a live vertical timeline (Reserved → Out → On Rent → Field calls → End Rent → Recovery → Returned) that reads top-to-bottom like a logbook. Notes for review:
  • Stage rows use the same colour = state function as everywhere else: green dot = done, filled yellow ring = current, hollow grey = upcoming — no new state vocabulary.
  • The NOW + next header is the one place text repeats the current stage in plain language; the actual Door lives once, on the current stage's row (Recovery here) — never duplicated as a second clickable button.
  • Route strips (Out, Recovery only — per spec) are two Refs and a distance Stamp, not a map.
  • The section-chip rail carries the other Units sections greyed (Inspection, Services, Work Orders, Specs, Investment, GPS) plus To Do — Yard Journey sits active among them, so this reads as one section of the larger inline-expand card, not a standalone screen.
  • Dormant units (Scout, right) get a calm Available Stamp + the one + Start rental Door — never an empty rail.
  • Matte throughout — no glow; the current-stage ring is a flat 3px outline, not a blur. Dimmed CVD-safe --yellow #eed44b used for "current," never the neon value.
