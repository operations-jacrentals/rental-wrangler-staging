The Dashboard card

Every role already keeps the same five cards — Units · Rentals · Customers · Trips · Categories. This adds a 6th: Dashboard, shaped to whoever's looking at it. The Sales card already is a dashboard — the sales job laid out as graphs — so this generalises that idea to every role. Two variants below (Sales and Mechanic) so the role-dependent point is concrete. Nothing here replaces a base card — Yard Journey stays on Units, routes stay on Trips; the charts only drill into those cards with a filter already applied.

First draft — react to it, not final. A few calls made to get this concrete:
  • Toggle defaults to Mechanic on load (not Sales) so the one fully-legible drill — the Down wedge callout — is visible without clicking anything.
  • The funnel and revenue-trend charts stay blue/neutral rather than status colour, since a pipeline stage or a dollar trend isn't a status enum — only charts that directly split a status field (account standing, fleet state) use the locked red/yellow/green/gray Signal palette. Worth confirming that split reads right to you.
  • "Selected" on the Down wedge is shown as a thicker stroke + a bordered legend row + a static callout, not a blurred glow (matte constraint) — real build would likely add this on hover/click, shown here baked in so it's visible at rest.
  • Click and double-click on every mark are wired to the log line at the bottom, so you can click around and see exactly what would filter vs. what would open a tab.
  • Work-order backlog drills to Units (that's where WOs live today) — flag if you want it split differently, e.g. its own view.
  • Not addressed here on purpose: which roles get which numbers at all (money visibility, etc.) — that's the separate keep-the-keys access-gate call, not a design decision.
